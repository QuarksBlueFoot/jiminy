## 1. Trait Declaration

### `Pod` marker trait - `crates/jiminy-core/src/account/pod.rs`

```rust
pub unsafe trait Pod: Copy + 'static {}
```

Sound because: implementing types must be `#[repr(C)]` or
`#[repr(transparent)]`, `Copy`, valid for all bit patterns, and have
no interior references. This is a contract, not an operation.

## 2. Unsafe Function

### `load_unchecked` - `crates/jiminy-core/src/account/overlay.rs`

Generated by `zero_copy_layout!`. Tier 3 load with no header
validation. Sound because the caller assumes responsibility for
ensuring the data represents a valid layout instance (documented via
`# Safety` on the generated method).

## 8. PDA Derivation

Four `unsafe` blocks in `jiminy-core/src/check/pda.rs`:

1. Writing seeds into `MaybeUninit` array. Loop index bounded by `N`.
2. Writing bump/program_id/marker. Index bounded by `N + 2`.
3. Calling `sol_sha256`. Valid input/output buffers.
4. `assume_init`. Buffer fully written by `sol_sha256`.

## Miri Coverage

All `jiminy-core` tests run under Miri in CI. This covers:

- `Pod` overlay round-trips
- `Le*` type construction and reads
- `FieldRef` / `FieldMut` slice operations
- `split_fields` / `split_fields_mut` borrow splitting
- `ZeroCopySlice` / `ZeroCopySliceMut` access patterns
- `SegmentSlice` / `SegmentSliceMut` access patterns
- `SegmentTable` / `SegmentTableMut` init and validation
- `AccountHeader` overlay and mutation

Miri validates that no undefined behavior (invalid pointer
dereferences, use-after-free, data races, etc.) occurs in these paths.
